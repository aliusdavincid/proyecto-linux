# S1M01-0001 - ¿Qué es Linux?

---

## 🎯 Objetivo del tema
Entender qué es Linux a nivel conceptual, cómo se compone (kernel vs sistema completo), su origen y por qué es la base de la infraestructura tecnológica moderna.

---

## 🧠 Conceptos clave

| Concepto | Explicación |
|----------|------------|
| Kernel | Núcleo del sistema operativo que gestiona hardware, memoria y procesos |
| Linux | Kernel creado por Linus Torvalds en 1991 |
| GNU/Linux | Sistema completo: kernel Linux + herramientas del proyecto GNU |
| Distribución | Sistema operativo basado en Linux con software adicional |
| Código abierto | Software cuyo código puede ser estudiado, modificado y distribuido libremente |

---

## 📖 Explicación

### 1. Contexto
A inicios de los años 90, los sistemas operativos eran principalmente propietarios, costosos o limitados. No existía una alternativa libre y potente para usuarios y desarrolladores.

### 2. Desarrollo
En 1991, Linus Torvalds desarrolló el kernel Linux como un proyecto personal y lo publicó en internet. Este kernel se combinó con herramientas del proyecto GNU, dando origen a un sistema operativo completo.

### 3. Resultado / impacto
Linux se convirtió en la base de gran parte de la infraestructura tecnológica global. Su modelo de desarrollo abierto permitió una adopción masiva en servidores, nube, dispositivos móviles y sistemas críticos.

---

## 🌍 Aplicación real

Linux está presente en:

- Servidores (Google, Amazon, Netflix)  
- Computación en la nube (AWS, Azure, GCP)  
- Android (basado en kernel Linux)  
- Supercomputadoras  
- Dispositivos embebidos (routers, TVs, IoT)  

---

## 🗂️ Distribuciones importantes

Las distribuciones son sistemas completos construidos sobre el kernel Linux. Cada una tiene un enfoque distinto.

### 🔹 Uso general / aprendizaje

- **Ubuntu**  
  La distribución más popular. Muy usada en cloud y desarrollo.

- **Debian**  
  Extremadamente estable. Base de muchas otras distribuciones.

- **Linux Mint**  
  Basada en Ubuntu. Ideal para empezar.

---

### 🔹 Entornos empresariales / servidores

- **Red Hat Enterprise Linux (RHEL)**  
  Estándar en empresas. Soporte comercial.

- **AlmaLinux / Rocky Linux**  
  Alternativas gratuitas compatibles con RHEL.

---

### 🔹 Avanzadas / control total

- **Arch Linux**  
  Minimalista. Instalación manual. Máximo control.

---

### 🔹 Especializadas

- **Kali Linux**  
  Orientada a ciberseguridad.

---

### 🧠 Idea clave

Todas usan el mismo kernel Linux.  
Lo que cambia es:

- gestor de paquetes  
- herramientas incluidas  
- nivel de control vs facilidad  

---

### ⚡ En la práctica

- Servidores → Debian / Ubuntu / RHEL-like  
- Cloud → Ubuntu y RHEL-based  
- Aprendizaje → Ubuntu / Mint

## 🧪 Ejemplo práctico (explorando Linux desde la terminal)


```bash
🔍 Ver información del kernel
uname -r
uname -a

🧾 Ver información del sistema operativo
cat /etc/os-release

🧠 Ver arquitectura del sistema
uname -m

📦 Ver distribución exacta
lsb_release -a

(Si no está disponible:)

sudo apt install lsb-release

🖥️ Ver información general del sistema
hostnamectl

📂 Ver estructura base del sistema
ls /

📌 Ver usuario actual
whoami

🧵 Ver procesos activos
ps aux | head

🔎 Ver detalles del kernel (útil en entrevistas)
uname -srm

💡 Ayuda de comandos
man uname
uname --help

```

### 💼 En entrevistas

Pregunta típica:
¿Qué es Linux?

### Respuesta sólida:

"Linux es un kernel de código abierto creado en 1991 por Linus Torvalds. Es la base de la mayoría de servidores y sistemas modernos. 

Se usa a través de distribuciones como Ubuntu, Debian o Red Hat, que forman sistemas operativos completos."

---

### ⚠️ Errores comunes

Pensar que Linux es un sistema operativo completo
No diferenciar kernel vs distribución
Creer que solo se usa en PCs
Memorizar teoría sin practicar en terminal
