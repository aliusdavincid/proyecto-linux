# S1M01-0001 — ¿Qué es Linux?

---

## Conceptos clave

| Término | Definición |
|---------|------------|
| Kernel | Núcleo del sistema operativo. Gestiona hardware, memoria y procesos |
| Linux | Kernel creado por Linus Torvalds en 1991 |
| GNU/Linux | Sistema completo: kernel Linux + herramientas del proyecto GNU |
| Distribución | Sistema operativo construido sobre el kernel Linux con software adicional |
| Código abierto | Software que puede ser estudiado, modificado y distribuido libremente |

---

## Lo que ejecuté

### En Debian GNU/Linux 13 (trixie) — TTY pura

```bash
# uname -r — muestra la versión exacta del kernel en ejecución
# lo importante: el número de versión y la arquitectura (amd64)
aliusdavincid@debian:~$ uname -r
6.12.74+deb13+1-amd64

# uname -a — muestra toda la información del sistema en una sola línea
# lo importante: nombre del host, versión del kernel, fecha de compilación, arquitectura
aliusdavincid@debian:~$ uname -a
Linux debian 6.12.74+deb13+1-amd64 #1 SMP PREEMPT_DYNAMIC Debian 6.12.74-2 (2026-03-08) x86_64 GNU/Linux

# uname -m — muestra la arquitectura del procesador
# lo importante: x86_64 significa sistema de 64 bits
aliusdavincid@debian:~$ uname -m
x86_64

# cat /etc/os-release — muestra información de la distribución instalada
# lo importante: NAME, VERSION y VERSION_CODENAME identifican la distro exacta
aliusdavincid@debian:~$ cat /etc/os-release
PRETTY_NAME="Debian GNU/Linux 13 (trixie)"
NAME="Debian GNU/Linux"
VERSION_ID="13"
VERSION="13 (trixie)"
VERSION_CODENAME=trixie
DEBIAN_VERSION_FULL=13.4
ID=debian

# hostnamectl — muestra información completa del sistema y el hardware
# lo importante: hostname, OS, kernel, arquitectura y modelo del equipo
aliusdavincid@debian:~$ hostnamectl
Static hostname: debian
      Icon name: computer-convertible
        Chassis: convertible
Operating System: Debian GNU/Linux 13 (trixie)
         Kernel: Linux 6.12.74+deb13+1-amd64
   Architecture: x86-64
Hardware Vendor: Lenovo
 Hardware Model: Lenovo ideapad FLEX 5-1570

# lsb_release -a — muestra información de la distribución en formato resumido
# lo importante: Distributor ID y Codename identifican la rama de la distro
aliusdavincid@debian:~$ lsb_release -a
Distributor ID: Debian
Description:    Debian GNU/Linux 13 (trixie)
Release:        13
Codename:       trixie

# ls / — lista los directorios del nivel raíz del sistema de archivos
# lo importante: estos directorios son la base de todo el sistema Linux
aliusdavincid@debian:~$ ls /
bin  boot  dev  etc  home  initrd.img  lib  lib64  lost+found
media  mnt  opt  proc  root  run  sbin  srv  sys  tmp  usr  var  vmlinuz

# whoami — muestra el nombre del usuario activo en la sesión actual
aliusdavincid@debian:~$ whoami
aliusdavincid
```

### En Linux Mint 22.3 (Zena) — terminal gráfica

```bash
# uname -r — misma función, kernel diferente al de Debian en el mismo hardware
aliusdavincid@aliusdavincid:~$ uname -r
6.17.0-19-generic

# uname -a — información completa del sistema
aliusdavincid@aliusdavincid:~$ uname -a
Linux aliusdavincid 6.17.0-19-generic #19~24.04.2-Ubuntu SMP PREEMPT_DYNAMIC Fri Mar 6 23:08:46 UTC 2025 x86_64 GNU/Linux

# cat /etc/os-release — ID_LIKE muestra de qué distribuciones desciende Mint
aliusdavincid@aliusdavincid:~$ cat /etc/os-release
NAME="Linux Mint"
VERSION="22.3 (Zena)"
ID=linuxmint
ID_LIKE="ubuntu debian"
PRETTY_NAME="Linux Mint 22.3"
VERSION_CODENAME=zena
UBUNTU_CODENAME=noble

# hostnamectl — en Mint no aparecen advertencias de acceso denegado
aliusdavincid@aliusdavincid:~$ hostnamectl
Static hostname: aliusdavincid
      Icon name: computer-convertible
        Chassis: convertible
Operating System: Linux Mint 22.3
         Kernel: Linux 6.17.0-19-generic
   Architecture: x86-64
Hardware Vendor: Lenovo
 Hardware Model: Lenovo ideapad FLEX 5-1570

# whoami — el usuario es el mismo en ambos sistemas
aliusdavincid@aliusdavincid:~$ whoami
aliusdavincid
```

## Lo que me pasó

**Error de tipeo en Debian:**
Escribí `hostnameclt` en lugar de `hostnamectl`. La terminal respondió:
```
bash: hostnameclt: orden no encontrada
```
La TTY no perdona errores de tipeo — no hay autocompletado visual ni corrección automática.

**Advertencias en `hostnamectl` sin sudo (Debian):**
```
Failed to query product UUID, ignoring: Access denied
Failed to query hardware serial, ignoring: Access denied
```
El comando funcionó igual, pero Debian restringió acceso a datos de hardware sin privilegios. En Mint el mismo comando no mostró ninguna advertencia.

**Kernels distintos en la misma máquina:**
Debian corre el kernel `6.12.74+deb13+1-amd64` y Mint corre `6.17.0-19-generic`. Es el mismo hardware — un Lenovo ideapad FLEX 5-1570 — pero cada distribución gestiona su propia versión del kernel.

## Para qué sirve esto en la práctica

| Comando | Situación real |
|---------|----------------|
| `uname -r` | Verificar si el kernel es compatible antes de instalar un driver o módulo |
| `uname -a` | Reportar información del sistema al abrir un ticket de soporte técnico |
| `cat /etc/os-release` | Confirmar la distro de un servidor antes de instalar paquetes |
| `hostnamectl` | Identificar una máquina remota al conectarse por SSH |
| `lsb_release -a` | Scripts de automatización que necesitan saber en qué distro están corriendo |
| `ls /` | Orientarse en un sistema desconocido — ver qué directorios existen en la raíz |
| `whoami` | Confirmar con qué usuario estás operando antes de ejecutar algo crítico |

---

## Lo que me quedó claro

Linux es solo el kernel. Lo que uso en Debian y en Mint son distribuciones distintas construidas sobre ese mismo kernel. El comando `uname -r` muestra exactamente qué versión del kernel está corriendo — y en la misma máquina pueden ser versiones diferentes dependiendo del sistema que arranques.
