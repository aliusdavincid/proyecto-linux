# S1M01-0002 — La terminal: qué es un shell y bash

---

## Conceptos clave

| Término | Definición |
|---------|------------|
| Terminal | Interfaz que permite escribir comandos y recibir respuestas del sistema |
| Shell | Programa que interpreta los comandos que escribes en la terminal |
| Bash | El shell más común en Linux — Bourne Again Shell |
| Variable de entorno | Valor almacenado en memoria que el sistema usa para su configuración |
| $PATH | Variable que indica al sistema dónde buscar los programas ejecutables |
| /etc/shells | Archivo que lista todos los shells válidos instalados en el sistema |

---

## Lo que ejecuté

### En Debian GNU/Linux 13 (trixie) — TTY pura

```bash
# echo $SHELL — muestra la ruta del shell activo en tu sesión
# lo importante: confirma que estás usando bash y dónde está instalado
aliusdavincid@debian:~$ echo $SHELL
/bin/bash

# echo $BASH_VERSION — muestra la versión exacta de bash en ejecución
# lo importante: útil para saber si ciertas funciones están disponibles
aliusdavincid@debian:~$ echo $BASH_VERSION
5.2.37(1)-release

# echo $0 — muestra el nombre del shell o script en ejecución actual
# lo importante: cuando dices "bash" confirma que estás en una sesión interactiva
aliusdavincid@debian:~$ echo $0
bash

# cat /etc/shells — lista todos los shells válidos instalados en el sistema
# lo importante: solo los shells listados aquí pueden usarse como shell de login
aliusdavincid@debian:~$ cat /etc/shells
# /etc/shells: valid login shells
/bin/sh
/usr/bin/sh
/bin/bash
/usr/bin/bash
/bin/rbash
/usr/bin/rbash
/usr/bin/dash
/usr/bin/tmux

# bash --version — muestra información completa de la versión de bash
# lo importante: versión, arquitectura y licencia
aliusdavincid@debian:~$ bash --version
GNU bash, versión 5.2.37(1)-release (x86_64-pc-linux-gnu)
Copyright (C) 2022 Free Software Foundation, Inc.
Licencia GPLv3+: GPL de GNU versión 3 o posterior

# grep aliusdavincid /etc/passwd — muestra la entrada del usuario en el sistema
# lo importante: el último campo (/bin/bash) indica qué shell usa ese usuario por defecto
aliusdavincid@debian:~$ grep aliusdavincid /etc/passwd
aliusdavincid:x:1000:1000:nexus,,,:/home/aliusdavincid:/bin/bash

# echo $HOME — muestra el directorio personal del usuario activo
aliusdavincid@debian:~$ echo $HOME
/home/aliusdavincid

# echo $USER — muestra el nombre del usuario activo
aliusdavincid@debian:~$ echo $USER
aliusdavincid

# echo $PATH — muestra los directorios donde el sistema busca programas ejecutables
# lo importante: cuando escribes un comando el sistema lo busca en cada uno de estos directorios en orden
aliusdavincid@debian:~$ echo $PATH
/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
```

### En Linux Mint 22.3 (Zena) — terminal gráfica

```bash
# echo $SHELL — mismo resultado que Debian, ambos usan bash por defecto
aliusdavincid@aliusdavincid:~$ echo $SHELL
/bin/bash

# echo $BASH_VERSION — Mint tiene una versión de bash ligeramente diferente
aliusdavincid@aliusdavincid:~$ echo $BASH_VERSION
5.2.21(1)-release

# echo $0 — mismo resultado en ambos sistemas
aliusdavincid@aliusdavincid:~$ echo $0
bash

# cat /etc/shells — Mint tiene los mismos shells que Debian en este caso
aliusdavincid@aliusdavincid:~$ cat /etc/shells
# /etc/shells: valid login shells
/bin/sh
/usr/bin/sh
/bin/bash
/usr/bin/bash
/bin/rbash
/usr/bin/rbash
/usr/bin/dash
/usr/bin/tmux

# bash --version — versión diferente a Debian, mismo formato de salida
aliusdavincid@aliusdavincid:~$ bash --version
GNU bash, versión 5.2.21(1)-release (x86_64-pc-linux-gnu)
Copyright (C) 2022 Free Software Foundation, Inc.

# grep aliusdavincid /etc/passwd — el shell asignado es el mismo: /bin/bash
aliusdavincid@aliusdavincid:~$ grep aliusdavincid /etc/passwd
aliusdavincid:x:1000:1000:aliusdavincid,,,:/home/aliusdavincid:/bin/bash

# echo $PATH — Mint tiene más directorios en PATH que Debian
# lo importante: incluye /snap/bin y rutas de .dotnet que Debian no tiene
aliusdavincid@aliusdavincid:~$ echo $PATH
/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:/usr/local/games:/snap/bin:/home/aliusdavincid/.dotnet/tools
```

## Lo que me pasó

**Error al escribir `bash --version` en Debian:**
Escribí `bash //version` con doble barra. La terminal respondió:
```
bash: //version: No existe el fichero o el directorio
```
Bash interpretó `//version` como una ruta de directorio, no como una opción. El símbolo correcto es `--` (dos guiones), no `/`.

**Error al escribir `echo $SHELL` en Mint:**
Escribí `cho $SHELL` sin la `e` inicial. La terminal respondió:
```
Orden «cho» no encontrada.
```
Mint fue más descriptiva que Debian en el error — sugirió paquetes similares como `who`, `echo` y otros. Debian solo dice `orden no encontrada`.

**El teclado en inglés en la TTY de Debian me costó trabajo:**
Estoy acostumbrado al teclado latinoamericano y en la TTY de Debian el layout es inglés. Símbolos como `$`, `-` y `--` no están donde los espero. Esto me forzó a cometer varios errores de tipeo durante la sesión. Es algo que hay que entrenar — trabajar con ambos layouts es inevitable en entornos reales.

**Las variables de entorno en mayúsculas:**
Al principio no entendí por qué `$SHELL`, `$PATH` y `$USER` van en mayúsculas. Investigué y es una convención — las variables de entorno del sistema siempre van en mayúsculas para distinguirlas de las variables definidas por el usuario, que suelen ir en minúsculas.

**`$PATH` diferente entre Debian y Mint:**
Debian tiene un `$PATH` más corto y minimalista. Mint incluye rutas adicionales como `/snap/bin` y `/home/aliusdavincid/.dotnet/tools`. Esto refleja que Mint tiene más software instalado por defecto.

## Para qué sirve esto en la práctica

| Comando | Situación real |
|---------|----------------|
| `echo $SHELL` | Confirmar qué shell estás usando antes de escribir un script |
| `echo $BASH_VERSION` | Verificar compatibilidad de funciones antes de ejecutar un script en otro servidor |
| `cat /etc/shells` | Saber qué shells están disponibles antes de cambiar el shell de un usuario |
| `grep usuario /etc/passwd` | Verificar qué shell tiene asignado un usuario en el sistema |
| `echo $PATH` | Diagnosticar por qué el sistema no encuentra un comando que instalaste |
| `echo $HOME` | Usado en scripts para referenciar el directorio del usuario sin escribir la ruta completa |

---

## Lo que no haría / Lo que sí usaría

- ✅ Usaría `echo $PATH` como primer diagnóstico cuando un comando instalado no es encontrado por el sistema
- ✅ Usaría `grep usuario /etc/passwd` para verificar rápidamente el shell asignado a cualquier usuario sin abrir archivos de configuración
- ❌ No cambiaría de bash a otro shell mientras estoy aprendiendo — la lógica de trabajo es la misma en todos pero los comandos específicos cambian. Primero dominar bash
- ❌ No memorizaría el contenido de `$PATH` — es más útil saber cómo leerlo e interpretarlo que memorizarlo

---

## Lo que me quedó claro

El shell es el intermediario entre lo que escribo y lo que hace el sistema. Bash es el shell por defecto en Debian y Mint, pero no es el único. Lo más importante de esta sesión fue entender que las variables de entorno como `$PATH`, `$SHELL` y `$HOME` no son comandos — son valores almacenados en memoria que controlan cómo se comporta el sistema. En entornos reales casi siempre se trabaja con configuraciones mínimas, así que aprender a leer estas variables es más valioso que explorar shells alternativos.
